# Hospital_Management_project:
This project focuses on analyzing a Hospital Management dataset by modeling multiple tables, cleaning data and generating meaningful business insights. The goal is to transform raw hospital data into an interactive dashboard for better decision-making.
## 1. Project Overview:

This project focuses on analyzing hospital operations data to uncover actionable insights related to patient trends, revenue performance, and treatment distribution.
The workflow includes data cleaning, transformation, and analysis using SQL, followed by building an interactive Power BI dashboard for visualization.

## 2. Objectives:
* Total Revenue by Doctor.
* patients by gender.
* appointments per doctor.
* which department is highest revenue.
* most common treatment.
* Analyze monthly revenue and appointment trends.
* Top cities by patients.

 ## 3. Tools & Technologies:
* Database: MySQL
* Visualization: Power BI
* Query Language: SQL
### Data Processing Workflow
1. Data Cleaning
2. Data Modeling
Joined multiple tables:
Patients
Appointments
Treatments
Doctors
Created a final dataset: hospital_management
3. Data Analysis
Used SQL for:
Aggregations (SUM, COUNT)
Joins
Group By & filtering
4. Create interactive Dashboard.

## 4. Key Analysis
* Patient count by city.
* Monthly revenue trends
* Appointment distribution by month
* Treatment frequency analysis
###  Business Insights:
* August generated the highest revenue, indicating peak hospital activity
* Maple City has the highest number of patients, showing strong demand in that region
* Appointments are highest in the 8th month, aligning with revenue growth
* Chemotherapy is not the most common treatment, suggesting other treatments dominate patient care
